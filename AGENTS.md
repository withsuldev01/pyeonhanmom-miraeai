# 프로젝트 작업 지침

## 프로젝트 개요

이 프로젝트는 `편한몸 의원 부설 미래아이 심리발달 클리닉`의 센터 홍보 랜딩 페이지를 만든다.

페이지의 핵심 목적은 아동 발달이 걱정되는 보호자가 센터의 전문성과 분위기를 빠르게 이해하고, 상담 신청 또는 전화상담으로 자연스럽게 이동하게 하는 것이다.

## 기준 문서와 자료

작업 전 다음 문서를 우선 확인한다.

- `docs/landing-page-brief.md`: 랜딩 페이지 기획 초안
- `docs/landing-page-wireframe.md`: 섹션 구조와 데스크톱/모바일 와이어프레임
- `docs/편한몸홈페이지.pdf`: 원본 제안/와이어프레임
- `docs/logo.png`: 센터 로고
- `docs/center-lobby.jpeg`: 센터 로비 실사 이미지

PDF는 비전문가가 전달한 참고안이므로 그대로 복제하지 않는다. 문서에 정리된 목적, CTA 동선, 브랜드 정리 방향을 우선한다.

## 주요 전환 동선

`상담 신청`, `상담 문의`, `초기 상담 평가지 작성하기` 성격의 CTA는 모두 아래 외부 URL로 이동한다.

```text
https://child-developmental-assessment-form-sage.vercel.app/
```

PDF 2쪽과 마지막 쪽의 상담 평가지 화면은 위 외부 페이지의 초기 화면 참고 이미지다. 랜딩 페이지 내부에 같은 폼을 직접 구현하지 않는다.

`전화상담`은 전화번호가 확정되기 전까지 임시 처리하거나 비활성/대체 안내로 둔다. 전화번호가 확정되면 `tel:` 링크를 사용한다.

## 브랜드와 문구

- 최종 브랜드 표기는 기본적으로 `편한몸 의원 부설 미래아이 심리발달 클리닉`을 사용한다.
- PDF 안의 `코아` 문구는 그대로 쓰지 말고 `미래아이` 또는 전체 센터명으로 정리한다.
- 보호자를 불안하게 만들거나 치료 효과를 보장하는 과장 문구를 피한다.
- 문장 톤은 전문적이되 차갑지 않게, 아이와 보호자를 함께 바라보는 안정감 있는 톤으로 작성한다.

## 디자인 방향

- 실제 센터 공간 이미지와 로고를 중심으로 신뢰감을 만든다.
- 장식이 과한 캐릭터, 과도한 그라디언트, 어두운 오버레이, 의미 없는 배경 장식은 피한다.
- 로고의 네이비, 청록, 노랑, 빨강, 초록 계열은 포인트 컬러로만 사용한다.
- 전체 배경은 흰색 또는 아주 옅은 웜그레이 계열을 기본으로 한다.
- 카드는 필요한 정보 묶음에만 사용하고, 중첩 카드 구조는 피한다.
- 카드 모서리는 특별한 이유가 없으면 8px 내외로 제한한다.

## 기기 지원 범위와 반응형 기준

기본 디자인 기준은 데스크톱 풀사이즈 화면이다.

다만 모바일도 반드시 지원한다. 반응형으로 전환했을 때 섹션 순서, CTA 위치, 이미지 크롭, 텍스트 줄바꿈이 어색하지 않아야 한다.

작업 시 최소한 다음 뷰포트를 고려한다.

- 데스크톱 풀사이즈: 1440px 이상
- 노트북: 1280px 전후
- 태블릿 또는 좁은 화면: 768px 전후
- 모바일: 375px 전후

반응형 규칙:

- 모바일에서는 히어로를 텍스트 먼저, 이미지 다음 순서로 배치한다.
- 모바일 첫 화면 안에 `상담 신청` CTA가 보여야 한다.
- 프로그램/카드형 섹션은 모바일에서 1열 또는 안정적인 2열로 전환한다.
- 치료 과정은 모바일에서 세로 타임라인을 기본으로 한다.
- 치료사 소개와 후기는 모바일에서 가로 스와이프 또는 1열 카드 흐름을 사용할 수 있다.
- 텍스트가 버튼, 카드, 이미지 위에서 겹치거나 잘리지 않도록 한다.
- 이미지가 의미 없는 부분만 보이도록 과하게 크롭되지 않게 한다.

## 구현 지침

- Next.js 코드를 작성하기 전에는 반드시 `node_modules/next/dist/docs/`의 관련 문서를 확인한다.
- 기존 App Router 구조와 프로젝트 파일 구성을 우선한다.
- 랜딩 페이지는 실제 사용 가능한 첫 화면으로 시작해야 하며, 별도 마케팅용 소개 페이지를 앞에 두지 않는다.
- 상담 신청 CTA 링크는 한 곳에서 상수화하거나 명확하게 관리해 모든 버튼이 같은 URL을 사용하게 한다.
- 시각 자료는 가능한 한 `docs/logo.png`, `docs/center-lobby.jpeg`를 우선 사용한다.
- 새 이미지나 일러스트가 필요하면 실제 섹션 목적에 맞는 자산을 사용하고, 임시 박스나 의미 없는 장식으로 대체하지 않는다.

## Motion 애니메이션 지침

- 애니메이션은 `motion` 라이브러리를 사용한다.
- `app/page.tsx` 전체를 Client Component로 바꾸지 말고, `app/components/Motion.tsx` 같은 전용 Client Component 래퍼에서만 Motion을 사용한다.
- 기본 톤은 의료/상담센터에 맞게 차분한 등장 애니메이션으로 유지한다.
- 주요 적용 범위는 히어로, 상담 안내, 카드 그리드, 패키지 CTA, 후기 섹션이다.
- 기본 패턴은 `opacity 0 -> 1`, `y 16~20 -> 0`, 필요 시 이미지 `scale 0.98 -> 1` 정도로 제한한다.
- 카드 그리드는 `staggerChildren`으로 순차 등장시키되 과한 지연은 피한다.
- CTA 버튼은 `whileHover`, `whileTap`으로 약한 반응만 준다.
- `useReducedMotion()`을 사용해 사용자의 동작 줄이기 설정을 존중한다.
- 번들 크기를 고려해 `LazyMotion`, `domAnimation`, `motion/react-m` 조합을 우선한다.

## 확인이 필요한 항목

아래 항목은 확정 전까지 임시값으로 처리하거나 사용자에게 확인한다.

- 전화상담 번호
- 치료사 소개 방식: 실제 인물 프로필, 분야별 팀 소개, 원장님 중심 소개 중 선택
- 패키지 구성과 가격 `60,000원` 최종 확정 여부
- 후기 문구의 실제 사용 가능 여부
- 센터 주소, 운영 시간, 지도 노출 여부
